METHOD 3: Personal Access Token (PAT)
================================================================================

Prerequisites:
  - GitHub Personal Access Token (PAT) generated
  - PAT must have 'repo' scope

Steps:

1. Update remote URL with PAT:
   git remote set-url origin https://[USERNAME]:[PAT]@github.com/kkendoh-ctrl/meisui-world.git
   
   Replace:
   - [USERNAME] with: kkendoh-ctrl
   - [PAT] with your actual token

2. Push to GitHub:
   git push -u origin main

3. (Optional) Save credentials:
   git config credential.helper store
   (This will cache credentials for future use)

================================================================================
TROUBLESHOOTING
================================================================================

Q: GitHub Pages shows 404 error
A: 1. Wait 5-10 minutes for initial build
   2. Check Settings → Pages shows "Your site is published"
   3. Clear browser cache (Ctrl+Shift+Delete)
   4. Try hard refresh (Ctrl+F5)

Q: Push fails with "fatal: could not read Username"
A: Use GitHub CLI (gh) which handles authentication automatically:
   gh auth login
   git push -u origin main

Q: Authentication fails
A: 1. Verify Personal Access Token has 'repo' scope
   2. Ensure token is not expired
   3. Try: git credential reject github.com
   4. Then try push again (will prompt for credentials)

Q: Files don't appear on GitHub
A: 1. Check "git status" shows clean working tree
   2. Verify commit was created: git log --oneline
   3. Ensure you're on 'main' branch: git branch
   4. Try: git push -f origin main (force push if needed)
