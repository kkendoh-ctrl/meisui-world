# テスト手順書

Firebase Realtime Database を使用したマルチデバイス投票システムの動作確認手順です。

## 事前準備

### ローカルテスト環境

```bash
# ローカルサーバーを起動（Python 3）
cd /sessions/stoic-funny-faraday/mnt/めいすいくん/github-deploy/
python -m http.server 8000

# または（Python 2）
python -m SimpleHTTPServer 8000

# または（Node.js）
npx http-server
```

ブラウザで以下にアクセス：
- http://localhost:8000/index.html

## テストシナリオ

### テスト1: 基本的なページ読み込み

**目的:** ページが正常に読み込まれることを確認

**手順:**
1. `vote.html` にアクセス
2. ページ全体が表示されることを確認
3. 6つのキャラクターカードが表示されることを確認
4. 投票数が「0」で初期化されていることを確認

**期待結果:**
- ✅ ページが正常に読み込まれる
- ✅ エラーがコンソールに表示されない
- ✅ すべてのキャラクターが表示される

### テスト2: Firebase 接続確認

**目的:** Firebase データベースへの接続を確認

**手順:**
1. `vote.html` の右上の接続ステータスを確認
2. 緑色の点が表示されていることを確認
3. 「接続中 ✓」と表示されていることを確認
4. ブラウザコンソール（F12）をチェック

**期待結果:**
- ✅ 接続ステータスが緑色
- ✅ テキストが「接続中 ✓」
- ✅ コンソールに Firebase エラーがない

### テスト3: 単一投票テスト

**目的:** 単一デバイスから投票できることを確認

**手順:**
1. `vote.html` を開く
2. イチョウめいすいくん（第1キャラ）の「投票する」ボタンをクリック
3. 「投票ありがとう！」メッセージが表示されることを確認
4. 2秒後にメッセージが消えることを確認
5. 投票数が「1」に更新されることを確認

**期待結果:**
- ✅ メッセージが表示される
- ✅ メッセージが2秒後に消える
- ✅ 投票数が「1」に更新される

### テスト4: リアルタイム同期テスト

**目的:** 複数デバイス間でのリアルタイム同期を確認

**手順:**
1. ブラウザタブAで `vote.html` を開く
2. ブラウザタブBで別のブラウザウィンドウで `vote.html` を開く
3. タブAでイチョウめいすいくんに投票（合計: 1票）
4. タブBでツツジめいちゃんに投票（合計: 1票）
5. タブAの投票数を確認
6. タブBの投票数を確認

**期待結果:**
- ✅ タブAのツツジめいちゃん投票数が「1」に更新される
- ✅ タブBのイチョウめいすいくん投票数が「1」に更新される
- ✅ 遅延は1秒以内

### テスト5: リアルタイム結果表示テスト

**目的:** `display.html` がリアルタイムで結果を表示することを確認

**手順:**
1. ブラウザAで `vote.html` を開く
2. ブラウザBで `display.html` を開く（別タブまたは別ウィンドウ）
3. ブラウザAでいくつかのキャラクターに投票（複数回）
4. ブラウザBの表示が更新されることを確認

**期待結果:**
- ✅ 投票直後にランキングが更新される
- ✅ グラフが更新される
- ✅ 総投票数が更新される
- ✅ 町ランクが計算される

### テスト6: 複数デバイス同時投票テスト

**目的:** 複数デバイスから同時投票できることを確認

**手順:**
1. 複数のブラウザウィンドウ/タブで `vote.html` を開く（最小3個）
2. 各デバイスで異なるキャラクターに投票
3. すべてのデバイスで投票数が更新されることを確認
4. `display.html` の順位も更新されることを確認

**期待結果:**
- ✅ すべての投票がカウントされる
- ✅ 競合なくトランザクションが成功する
- ✅ 最終的なカウントが正しい

### テスト7: リセット機能テスト

**目的:** リセット機能が正常に動作することを確認

**手順:**
1. `vote.html` で複数のキャラクターに投票
2. 右下の「0票に戻す」ボタンをクリック
3. 確認ダイアログが表示されることを確認
4. 「OK」をクリック
5. すべての投票数が「0」に戻ることを確認

**期待結果:**
- ✅ 確認ダイアログが表示される
- ✅ すべての投票数が「0」になる
- ✅ `display.html` の表示も「0」に更新される

### テスト8: レスポンシブデザインテスト

**目的:** 異なる画面サイズに対応することを確認

**手順:**
1. `vote.html` をブラウザで開く
2. F12 で開発者ツールを開く
3. デバイスエミュレーションを有効化（Ctrl+Shift+M）
4. 異なるデバイスで表示をテスト：
   - iPad (768x1024)
   - iPad Pro (1024x1366)
   - iPhone 12 (390x844)
5. ボタンが適切に配置されることを確認

**期待結果:**
- ✅ ボタンが80px以上の高さを保つ
- ✅ テキストが読みやすい
- ✅ タップターゲットが十分な大きさ

### テスト9: アニメーションテスト

**目的:** display.html のアニメーションが正常に動作することを確認

**手順:**
1. `display.html` をフルスクリーン表示
2. 以下のアニメーションを確認：
   - 浮遊するキャラクターアイコン
   - 桜のパーティクル落下
   - 雲のフローティング
   - 昼夜サイクルの変化
3. 投票時のフラッシュアニメーションを確認

**期待結果:**
- ✅ すべてのアニメーションがスムーズ
- ✅ フレームレートが安定している（30fps以上）
- ✅ CPU 使用率が過度に高くない

### テスト10: 町ランク計算テスト

**目的:** 町ランクが正しく計算されることを確認

**手順:**
1. `vote.html` と `display.html` を開く
2. 異なる投票数で町ランクの変化を確認

**投票数別の期待ランク:**
| 投票率 | ランク | 色 |
|---|---|---|
| 0～15% | F | グレー |
| 15～30% | E | パープル |
| 30～45% | D | グリーン |
| 45～60% | C | ブルー |
| 60～75% | B | パープル |
| 75～90% | A | レッド |
| 90～100% | S | オレンジ |

### テスト11: ネットワーク遅延シミュレーション

**目的:** ネットワーク遅延下での動作を確認

**手順:**
1. ブラウザの開発者ツール（F12）を開く
2. Network タブを開く
3. "Slow 3G" または "Fast 3G" を選択
4. `vote.html` をリロード
5. 投票してみる
6. 接続状態表示を確認

**期待結果:**
- ✅ ページが読み込まれる（遅い可能性あり）
- ✅ 投票が失敗せず成功する
- ✅ 接続状態が正しく表示される

### テスト12: オフライン→オンライン復帰テスト

**目的:** ネットワーク復帰後の自動同期を確認

**手順:**
1. Chrome 開発者ツール（F12）を開く
2. Network タブを開く
3. "Offline" を選択
4. `vote.html` で投票を試みる
5. 接続状態が赤色になることを確認
6. "Offline" をオフにしてネットワークを復帰
7. 接続状態が緑色に戻ることを確認
8. 投票数が同期されることを確認

**期待結果:**
- ✅ オフライン時に接続状態が赤色
- ✅ ネットワーク復帰後に接続状態が緑色
- ✅ 投票が適切に同期される

```javascript
// コンソール（F12）で実行
// 100件の投票をシミュレート
(async () => {
  for (let i = 0; i < 100; i++) {
    const charKey = ['meisui', 'tsutsuji', 'meisui_chan', 'bekabune', 'asari', 'ryoushi'][Math.floor(Math.random() * 6)];
    const voteRef = firebase.database().ref(`votes/${charKey}`);
    await voteRef.transaction(current => (current || 0) + 1);

  }
  console.log('テスト完了');
})();
```

## 既知の制限事項

1. **スケーラビリティ:** 大量の並行投票時にレイテンシが増加
2. **セキュリティ:** デフォルトではセキュリティルールが無設定
3. **ブラウザ:** 古い Internet Explorer には非対応
4. **オフライン投票:** オフライン中の投票はローカルキャッシュされない
